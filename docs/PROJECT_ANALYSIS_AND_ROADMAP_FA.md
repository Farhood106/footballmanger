تحلیل پروژه و نقشه راه تکمیل Football Manager آنلاین

## 1) برداشت محصول (بر اساس نیاز شما)

هدف اصلی: ساخت یک **شبیه‌ساز مدیریت/مالکیت باشگاه فوتبال آنلاین چندنفره** که:

- هم برای **مربی** حس اثرگذاری واقعی روی نتیجه بسازد.
- هم برای **مالک** حس مدیریت واقعی کسب‌وکار باشگاه ایجاد کند.
- اگر باشگاهی مالک یا مربی نداشت، با **AI** اداره شود تا چرخه بازی متوقف نشود.
- اختلاف مالک و مربی در یک **کمیسیون داوری داخلی** بررسی شود.
- مدل اقتصادی بازی (اسپانسر، قرارداد، تبلیغات) در آینده به **درآمدزایی کاربران** برسد.

---

## 2) وضعیت فعلی کد (آنچه الان دارید)

### نقاط قوت

- معماری MVC ساده و مناسب برای شروع:
  - کنترلرها: `app/Controllers`
  - مدل‌ها: `app/Models`
  - سرویس‌ها: `app/Services`
- موتور مسابقه دارید (`MatchEngine`) و از xG + پوآسون + رخدادها استفاده می‌کند.
- مدل‌های اصلی مثل کاربر، باشگاه، فصل، مسابقه، نقل‌وانتقالات وجود دارند.
- مسیریابی و احراز هویت پایه برقرار است.

### شکاف‌های مهم نسبت به هدف نهایی شما

1. سیستم نقش‌های پیشرفته (مالک/مربی/AI) در سطح قرارداد و حقوق/تعهدات کامل نشده.
2. چرخه روزانه چند فازه (تمرین/بازی اول/تمرین دوم/بازی دوم) به‌صورت رسمی و قابل کنترل پیاده‌سازی نشده.
3. ماژول داوری اختلافات (کمیسیون) وجود ندارد.
4. اقتصاد واقعی باشگاه (اسپانسر، پرداخت قرارداد، فسخ و جریمه) کامل نیست.
5. مقیاس‌پذیری لیگ‌ها/دسته‌ها/سهمیه لیگ قهرمانان هنوز در حد طراحی اولیه است.

---

## 3) تغییر انجام‌شده در این مرحله

برای نزدیک شدن به نیاز شما، یک لایه رسمی برای مدیریت چرخه روزانه اضافه شد:

- سرویس جدید `GameCycleService` اضافه شد.
- تنظیمات چرخه روزانه (برای تیم‌های تک‌بازی و دو‌بازی در روز) در `config/config.php` اضافه شد.

این یعنی از این به بعد می‌توانیم در کد، فاز فعلی بازی را دقیق تشخیص بدهیم (مثلاً: «تنظیم ترکیب دوم»، «بازی دوم زنده»، «تمرین برای فردا») و بر همان اساس UI/قفل عملیات/اجرای شبیه‌سازی را کنترل کنیم.

---

## 4) طراحی پیشنهادی مرحله‌ای برای رسیدن به نسخه قابل عرضه

## فاز A — تثبیت هسته گیم‌پلی آنلاین (MVP واقعی)

1. **Role & Ownership Service**
   - قوانین مالک/مربی/AI را متمرکز کنید.
   - هر باشگاه همیشه یک حالت مدیریتی معتبر داشته باشد:
     - Owner + Manager
     - Owner only
     - AI only

2. **Daily Scheduler Worker**
   - یک job runner زمان‌بندی‌شده بسازید (cron هر 1 تا 5 دقیقه).
   - با `GameCycleService` تشخیص بدهد چه رویدادی باید اجرا شود (lock lineup, simulate match, apply training).

3. **Matchday State Snapshot**
   - قبل/بعد هر بازی snapshot ذخیره شود تا مربی اثر تصمیم را ببیند:
     - fatigue, morale, injuries, tactical cohesion

4. **خطاگیری و Audit Log**
   - تمام تغییرات مهم (اخراج مربی، تغییر بودجه، قرارداد) در لاگ غیرقابل‌حذف ثبت شود.

## فاز B — عمق مدیریتی و قراردادها

1. **قرارداد مالک-مربی**
   - مدت، حقوق، بند فسخ، جریمه، شرایط توافقی.
2. **Dispute / کمیسیون بررسی**
   - ثبت پرونده، مستندات، رأی، جریمه.
3. **Financial Ledger**
   - دفتر مالی استاندارد باشگاه:
     - درآمد: اسپانسر، پاداش مسابقات، فروش بازیکن
     - هزینه: حقوق، پاداش، فسخ، استعدادیابی

## فاز C — ساختار رقابت پویا

1. لیگ‌های متعدد، هر کدام با دسته‌های مختلف.
2. سهمیه لیگ قهرمانان قابل تنظیم برای هر لیگ.
3. امکان ایجاد لیگ/دسته جدید توسط ادمین بدون تغییر کد هسته.

---

## 5) پیشنهاد دیتابیس برای نیازهای کلیدی شما

حداقل جدول‌های جدید پیشنهادی:

- `club_ownership_offers`
- `manager_contracts`
- `club_governance_cases` (پرونده اختلاف)
- `club_governance_decisions`
- `club_finance_ledger`
- `daily_cycle_snapshots`
- `scheduler_jobs`

---

## 6) پیشنهاد فنی برای سبک‌بودن سرور (طبق دغدغه شما)

- شبیه‌سازی full-match فقط در زمان لازم اجرا شود، بقیه زمان فقط state transition.
- از queue سبک (Redis + worker تک‌نخی) شروع کنید.
- گزارش مسابقه را cache کنید؛ تولید مجدد نکنید.
- پردازش‌های سنگین مثل آنالیتیکس را asynchronous کنید.
- برای هر باشگاه فقط delta state ذخیره شود، نه بازنویسی کامل.

---

## 7) قدم بعدی پیشنهادی (عملی و نزدیک)

در گام بعدی بهتر است این 3 مورد را پیاده کنیم تا پروژه سریع وارد فاز قابل‌بازی شود:

1. `DailySchedulerCommand` (CLI) برای اجرای خودکار فازها.
2. `ClubGovernanceModel + Controller` برای اختلاف مالک/مربی.
3. `ManagerContractModel` برای اخراج/فسخ/پرداخت حق‌وحقوق.

این سه مورد دقیقاً همان نقطه‌ای است که بازی شما را از «دموی فوتبال منیجر» به «پلتفرم واقعی مدیریت باشگاه آنلاین» تبدیل می‌کند.
